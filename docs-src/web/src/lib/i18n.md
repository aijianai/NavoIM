# i18n.ts — React i18n Hooks

## Purpose

Provides React hooks and helper functions for internationalization, wrapping `@navo/shared`'s `t()` function with the store's language preference.

## Exports

| Export | Kind | Description |
|--------|------|-------------|
| `useT()` | Hook | Returns a translation function bound to the current language |
| `useLanguage()` | Hook | Returns the current `Language` value |
| `setLanguage(lang)` | Function | Updates language in the store (non-hook usage) |
| `getT()` | Function | Returns a translation function using the store's current language (non-hook, for use outside React) |
| `LANGUAGES` | Re-export | Available language constants |
| `detectBrowserLanguage` | Re-export | Browser language detection |

## Key Logic

**`useT()`** subscribes to `useChatStore.language` and returns a curried `sharedT(lang, key, params)`. Re-renders only when language changes.

**`getT()`** reads language via `useChatStore.getState()` (imperative, no subscription). Used in module-scope code and timeout callbacks where hooks are unavailable.

**`setLanguage(lang)`** delegates to `useChatStore.getState().setLanguage(lang)`, which persists to `localStorage`.

## Dependencies

| Import | Purpose |
|--------|---------|
| `@navo/shared` | `t`, `LANGUAGES`, `detectBrowserLanguage`, types |
| `./store` | `useChatStore` for language state |

## Constraints and Gotchas

- `getT()` captures language at call time, not at render time. If language changes between calls, each call uses the latest value.
- `useT()` returns a new function reference on every language change but the same reference otherwise (due to Zustand selector stability).
- Language preference is persisted to `localStorage` and also synced from the server during `hydrate`.

## Interactions

- **Store (`store.ts`):** Calls `getT()` at module scope for use in timeout callbacks.
- **API (`api.ts`):** Calls `getT()` at module scope for localized error messages.
- **Call controller (`call.ts`):** Calls `getT()` at module scope for error messages in async contexts.
- **Components:** Use `useT()` for reactive translations in JSX.
