# i18n.ts -- Internationalization

## Purpose

Provides internationalization (i18n) infrastructure for the entire application. Defines the supported languages, browser language detection, translation key type system, and all translation strings in three languages (zh-CN, en, ja).

## Exports

| Export | Kind | Description |
|--------|------|-------------|
| `Language` | Type | Union type of supported locale codes: `"zh-CN"`, `"en"`, `"ja"` |
| `LANGUAGES` | Constant | Array of `{ value, label }` for UI language selectors |
| `detectBrowserLanguage()` | Function | Returns the browser's preferred language, defaulting to `"zh-CN"` |
| `TranslationKey` | Type | Exhaustive union of all ~1088 translation key strings |
| `Translations` | Type | `Record<TranslationKey, string>` -- a complete translation dictionary |
| `t(lang, key, params?)` | Function | Looks up a translated string by language and key, with optional interpolation |
| `getT(lang)` | Function | Returns a curried `t` function bound to a specific language |
| `translationExists(key)` | Function | Type guard that checks whether a key exists in the translation map |

## Key Logic

**Language detection** (`detectBrowserLanguage`): Reads `navigator.language` (or `navigator.userLanguage` for older IE). Matches `"ja"` prefix for Japanese, `"en"` prefix for English, falls back to `"zh-CN"`.

**Translation lookup** (`t`): Performs a two-step lookup: (1) find the language dictionary, (2) find the key within it. If the key is missing, returns the key itself as a fallback. Supports parameter interpolation via `{param}` placeholders in the string (e.g., `"time.minutesAgo": "{n} min ago"`).

**Key categories** are organized by dot-separated namespace:
- `app.*` -- Application name, tagline, connection states
- `login.*` -- Auth screens (login, register, banned, maintenance)
- `common.*` -- Shared UI labels (save, cancel, confirm, etc.)
- `nav.*` -- Navigation items
- `friends.*` -- Friend management
- `chat.*` -- Chat UI
- `composer.*` -- Message composer
- `profile.*` -- User profile settings
- `channel.*` -- Channel management
- `admin.*` / `adminSettings.*` -- Admin panel
- `server.*` -- Server-side error messages (mirrors backend error codes)
- `message.*` -- Message display and actions
- `call.*` -- WebRTC call UI
- `media.*` -- Media search/gallery
- `location.*` -- Location sharing
- `translation.*` -- Translation settings

## Dependencies

None -- this is a pure TypeScript module with no external imports.

## Constraints and Gotchas

- The translation key type (`TranslationKey`) is a manually maintained union literal. Adding a new key requires updating both the type and all three language dictionaries (`zhCN`, `en`, `ja`).
- The `t()` function falls back to the raw key string when a translation is missing, which makes missing translations visible in the UI but can look unpolished.
- Server error messages (`server.*`) are defined in both `@navo/shared` (for server-side use) and in the web translations (for client-side display). The keys must stay in sync manually.
- `detectBrowserLanguage()` runs on the client only; it returns `"zh-CN"` when `navigator` is undefined (SSR/Node context).
- The `ja` (Japanese) dictionary is a subset of `zhCN`/`en` -- not all keys have Japanese translations yet.

## Interactions

- **Web `src/lib/i18n.ts`**: Re-exports and wraps `t` from `@navo/shared` with a React hook (`useI18n`) for component usage.
- **Web `src/lib/utils.ts`**: Uses `t()` and `detectCurrentLang()` for time formatting and message previews.
- **Server**: Uses `@navo/shared` types and translation strings for error responses.
- **Admin settings**: Translation keys are used throughout the admin panel UI components.
