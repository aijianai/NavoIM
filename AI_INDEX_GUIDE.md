# AI 索引更新指南

## 概述

本文件指导 AI 助手如何自动更新项目索引文件 (`INDEX.md`)。索引文件用于帮助 AI 快速理解项目结构和代码内容，提高代码分析和生成的效率。

## 何时需要更新索引

### 必须更新的情况

1. **添加新文件**
   - 创建新的源代码文件
   - 添加新的配置文件
   - 引入新的依赖

2. **删除文件**
   - 删除源代码文件
   - 移除配置文件
   - 删除不再使用的文件

3. **重大重构**
   - 文件重命名
   - 目录结构调整
   - 模块重新组织

4. **功能变更**
   - 添加新的 API 端点
   - 添加新的数据库表
   - 添加新的组件或模块

### 建议更新的情况

1. **文件大小变化超过 20%**
2. **新增重要函数或类**
3. **修改核心业务逻辑**
4. **更新技术栈或依赖**

## 索引更新流程

### 步骤 1: 分析当前状态

在更新索引之前，先分析当前项目状态：

```bash
# 查看文件变化
git status

# 查看具体变化
git diff

# 查看最近的提交
git log --oneline -10
```

### 步骤 2: 识别需要更新的部分

根据变化类型，确定需要更新的索引部分：

- **文件变化**: 更新 "核心文件索引" 部分
- **目录变化**: 更新 "目录结构" 部分
- **功能变化**: 更新 "代码结构摘要" 部分
- **API 变化**: 更新 "API 端点" 部分
- **数据库变化**: 更新 "数据库表结构" 部分

### 步骤 3: 更新索引文件

#### 更新文件列表

对于新增或删除的文件，更新对应的部分：

```markdown
### 前端组件 (web/src/components/)

| 文件 | 大小 | 功能描述 |
|------|------|----------|
| 新文件.tsx | 10KB | 新功能描述 |
| ... | ... | ... |
```

#### 更新目录结构

如果目录结构发生变化，更新目录树：

```
www/study_tool/
├── 新目录/
│   └── 新文件.ts
├── ... (其他目录保持不变)
```

#### 更新代码摘要

如果功能发生变化，更新代码结构摘要：

```markdown
### 服务端主要模块

1. **新模块** (`新模块.ts`)
   - 功能描述
   - 主要方法
```

### 步骤 4: 验证更新

更新完成后，验证索引的准确性：

1. 检查文件路径是否正确
2. 检查文件大小是否准确
3. 检查功能描述是否清晰
4. 检查格式是否一致

## 索引格式规范

### 文件信息格式

```markdown
| 文件名 | 大小 | 功能描述 |
|--------|------|----------|
| 文件名.ts | 10KB | 简短的功能描述 |
```

### 目录结构格式

```
目录名/
├── 子目录/
│   └── 文件名
├── 文件名
└── ... (其他文件)
```

### 代码摘要格式

```markdown
### 模块名称

1. **子模块** (`文件名.ts`)
   - 功能点 1
   - 功能点 2
   - 功能点 3
```

## 示例：添加新组件

假设我们添加了一个新的 React 组件 `NewComponent.tsx`：

### 1. 更新文件列表

在 "前端组件 (web/src/components/)" 部分添加：

```markdown
| NewComponent.tsx | 5KB | 新组件功能描述 |
```

### 2. 更新目录结构

在目录树中添加：

```
web/src/components/
├── NewComponent.tsx
├── ... (其他组件)
```

### 3. 更新代码摘要

如果这是一个重要的组件，在 "前端主要模块" 部分添加描述。

## 示例：添加新 API

假设我们添加了一个新的 API 端点 `GET /api/new-endpoint`：

### 1. 更新 API 端点

在 "API 端点" 部分添加：

```markdown
### 新功能 API
- `GET /api/new-endpoint` - 新功能描述
```

### 2. 更新代码摘要

如果这是一个重要的 API，在 "服务端主要模块" 部分更新相关模块的描述。

## 注意事项

1. **保持简洁**: 功能描述应该简短明了，避免冗长
2. **保持一致**: 使用一致的格式和风格
3. **保持准确**: 确保所有信息都是准确的
4. **保持及时**: 及时更新索引，避免过期信息

## 索引文件结构

```
INDEX.md
├── 项目概述
├── 目录结构
├── 核心文件索引
│   ├── 共享模块
│   ├── 服务端
│   ├── 前端组件
│   └── 前端库
├── 代码结构摘要
│   ├── 服务端主要模块
│   └── 前端主要模块
├── 数据库表结构
├── API 端点
├── 配置文件
└── 更新日志
```

## 自动更新建议

AI 助手可以在以下情况下自动更新索引：

1. **检测到文件变化**: 当检测到文件创建、删除或修改时
2. **用户请求时**: 当用户请求更新索引时
3. **任务完成后**: 在完成重要的代码修改任务后
4. **定期检查**: 定期检查项目状态并更新索引

## 索引验证

更新索引后，应该验证：

1. **完整性**: 所有重要文件都已包含
2. **准确性**: 文件信息（大小、描述）准确
3. **一致性**: 格式和风格一致
4. **时效性**: 信息是最新的

## 索引维护

定期维护索引：

1. **删除过期信息**: 删除不再相关的文件信息
2. **更新过时描述**: 更新不再准确的功能描述
3. **优化格式**: 优化索引格式，提高可读性
4. **添加新信息**: 添加新发现的重要信息
